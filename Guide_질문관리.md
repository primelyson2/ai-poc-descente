# 질문관리 가이드 — Select AI Test - Table list

`Select AI Test - Table list` 화면에서 **질문과 조회할 컬럼을 관리하는 방식**을 설명합니다. 화면 상단의 **질문관리방식** 라디오로 두 가지 중 하나를 선택합니다.

| 방식 | 저장 위치 | 공유 | 용도 |
|---|---|---|---|
| **local storage** | 브라우저(사용자별) | ✕ 개인 전용 | 각자 자주 쓰는 질문/컬럼을 빠르게 저장 |
| **DB** | Oracle DB 테이블(접속 스키마) | ✅ 같은 DB 사용자 공유 | 검증된 좋은 질문·조회컬럼 세트를 공유·재사용 |

> 선택한 방식은 **접속 DB별로 기억**되어, 다음 방문 시 자동으로 복원됩니다.

---

## 1. local storage 방식 (기본)

기존과 동일합니다. 화면의 **질문 / 조회할 컬럼 / 정렬기준** 입력란을 직접 쓰고, 각 입력란의 **저장 콤보(제목 + 추가/수정/삭제 + 저장된 목록)** 로 개인 브라우저(localStorage)에 저장·재사용합니다.

- 저장 데이터는 **그 브라우저·그 사용자**에게만 남습니다(다른 사람과 공유되지 않음).
- DB에 아무것도 만들지 않습니다.

---

## 2. DB 방식 (공유)

질문과 조회컬럼을 **Oracle DB 테이블에 등록**해 두고, **검색·선택**해서 쓰는 방식입니다. 같은 DB에 접속하는 모든 사용자가 동일한 질문·컬럼 목록을 공유합니다.

DB 방식을 선택하면 화면의 localStorage 입력란(질문/조회컬럼/정렬기준)이 사라지고, **[질문 선택]** 버튼과 선택 결과 요약이 표시됩니다.

### 2.1 전체 흐름

```
[질문 선택] 버튼
   │
   ▼
질문 선택 빌더 (질문 + 조회할 컬럼)
   ├─ [질문 검색] ─▶ 질문 검색 팝업 (부분일치 검색 / + 새 질문 등록)
   └─ [컬럼 선택] ─▶ 컬럼 선택 팝업 (질문별 컬럼 / 모든 조회칼럼 / + 컬럼 추가 / 관련성 평가)
   │
   ▼
[선택완료] ─▶ 화면의 질문·조회할 컬럼에 반영 ─▶ [실행]
```

### 2.2 질문 선택 (검색 · 등록)

**[질문 검색]** 을 누르면 질문 검색 팝업이 뜹니다.

- **부분일치 검색**: 검색어를 입력하면 질문 내용에 그 단어가 포함된 질문을 목록으로 보여줍니다(text like). 행을 클릭하면 선택됩니다.
- **새 질문 등록**: 검색란에 새 질문 내용을 적고 **[+ 새 질문 등록]** 을 누르면 DB에 저장되고 곧바로 선택됩니다. (테이블이 비어 있어도 여기서 처음 등록할 수 있습니다.)

> 빌더의 질문 입력란에 **직접 타이핑**한 뒤 [선택완료]/[컬럼 선택]을 하면, 그 질문이 **새 질문으로 DB에 저장**됩니다(추가 + 선택).

### 2.3 컬럼 선택 (질문별 · 전체 · 추가)

**[컬럼 선택]** 을 누르면 컬럼 선택 팝업이 뜹니다.

- **질문별 컬럼(기본)**: 현재 선택한 질문에 연결된 조회 컬럼만 체크박스로 보여줍니다.
- **모든 조회칼럼**: 체크박스를 켜면 **질문과 무관하게 DB에 등록된 모든 조회 컬럼**을 보여줍니다. 여기서 필요한 컬럼을 골라 담을 수 있습니다.
- **컬럼 추가**: 새 컬럼명을 입력하고 **[추가]** 하면 현재 질문에 연결되어 DB에 저장됩니다.

### 2.4 관련성 평가 (AI)

**"모든 조회칼럼"** 모드에서 컬럼을 선택하면 **[관련성 평가]** 버튼이 활성화됩니다. 누르면 AI가 **선택한 컬럼이 질문에 답하기에 적절한 조회 컬럼인지** 평가합니다.

- 평가에는 **현재 Chat설정의 AI Profile** 과 그 프로파일의 **스키마 컨텍스트(테이블/컬럼 Comment·Annotation 포함)** 가 사용됩니다.
- 결과는 **컬럼별 판정(관련 / 무관)과 사유**, 그리고 총평으로 표시됩니다.

예시:

| 컬럼 | 판정 | 사유 |
|---|---|---|
| 브랜드명 | 관련 | '브랜드별' 매출을 요구하므로 그룹화·식별에 필요 |
| 판매금액 | 관련 | '매출' 상위를 요구하므로 집계·비교에 필요 |
| 고객주소 | 무관 | 브랜드별 매출 순위 분석에는 불필요 |

> 평가는 **선택을 강제하지 않습니다.** 참고용 근거이며, 최종 선택은 사용자가 합니다.

### 2.5 선택완료 → 실행

**[선택완료]** 를 누르면 선택한 질문과 조회할 컬럼이 화면에 반영됩니다. 이후 **[실행]** 을 누르면 local storage 방식과 동일하게 SQL 생성·실행이 진행됩니다. (DB 방식에는 정렬기준이 없습니다.)

---

## 3. DB 테이블 구조

DB 방식은 접속 스키마에 아래 두 테이블을 **자동 생성**해 사용합니다(최초 사용 시 `CREATE TABLE IF NOT EXISTS` 로 멱등 생성).

### T_NL2SQL_QUESTION — 질문
| 컬럼 | 타입 | 설명 |
|---|---|---|
| `ID` | NUMBER (IDENTITY, PK) | 질문 식별자 |
| `QUESTION` | CLOB | 질문 내용 |
| `CREATED_AT` | TIMESTAMP | 등록 시각 |

### T_NL2SQL_COLUMN — 조회 컬럼
| 컬럼 | 타입 | 설명 |
|---|---|---|
| `ID` | NUMBER (IDENTITY, PK) | 컬럼 식별자 |
| `QUESTION_ID` | NUMBER | 연결된 질문(`T_NL2SQL_QUESTION.ID`) |
| `COLUMN_NAME` | VARCHAR2(400) | 조회 컬럼명 |
| `CREATED_AT` | TIMESTAMP | 등록 시각 |

- **질문 : 컬럼 = 1 : N** — 한 질문에 여러 조회 컬럼을 연결합니다.
- **"모든 조회칼럼"** 은 `T_NL2SQL_COLUMN` 전체의 **distinct 컬럼명**을 보여줍니다(질문 무관).

### 생성 스크립트

화면에서 DB 방식을 처음 사용할 때 아래 스크립트가 **자동 실행**되어 테이블을 만듭니다(이미 있으면 건너뜀). 관리자가 미리 만들어 두려면 접속 스키마에서 그대로 실행하면 됩니다.

```sql
-- 1. 질문 테이블
CREATE TABLE IF NOT EXISTS T_NL2SQL_QUESTION (
  id         NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  question   CLOB,
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP
);

-- 2. 조회 컬럼 테이블 (질문에 1:N 연결)
CREATE TABLE IF NOT EXISTS T_NL2SQL_COLUMN (
  id          NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  question_id NUMBER,                 -- T_NL2SQL_QUESTION.ID 참조(논리 FK)
  column_name VARCHAR2(400),
  created_at  TIMESTAMP DEFAULT SYSTIMESTAMP
);
```

> - `CREATE TABLE IF NOT EXISTS` 는 Oracle 23ai 문법입니다. 하위 버전이면 `IF NOT EXISTS` 를 빼고 실행하세요(이미 있으면 `ORA-00955`).
> - `GENERATED BY DEFAULT AS IDENTITY` 로 `ID` 가 자동 채번됩니다(별도 시퀀스 불필요).
> - `QUESTION_ID` 는 논리적 FK 입니다(제약조건 없이 애플리케이션에서 연결). 무결성 제약이 필요하면 `FOREIGN KEY (question_id) REFERENCES T_NL2SQL_QUESTION(id)` 를 추가할 수 있습니다.
> - 테이블 생성에는 접속 계정의 **`CREATE TABLE`** 권한이 필요합니다.

---

## 4. 참고 · 주의

- **권한**: DB 방식의 테이블 자동 생성에는 접속 계정의 **`CREATE TABLE`** 권한이 필요합니다. 없으면 화면에 오류가 표시됩니다.
- **공유 범위**: 질문·컬럼은 **그 DB에 접속하는 모든 사용자**가 공유합니다. 헤더에서 DB를 바꾸면 그 DB의 목록으로 전환됩니다(질문/컬럼은 각 DB 테이블에 저장되므로 DB별로 분리).
- **평가 프로파일**: 관련성 평가는 화면 상단 **Chat설정에서 선택한 AI Profile** 을 사용합니다. Profile 이 없으면 평가할 수 없습니다.
- **정렬기준**: local storage 방식에만 있습니다. DB 방식은 질문 + 조회할 컬럼만 관리합니다.
- **개인 저장과 공유의 병행**: 두 방식은 독립적입니다. 상단 라디오로 언제든 전환할 수 있으며, 서로의 데이터에 영향을 주지 않습니다.
