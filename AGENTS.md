# Real ASTA Codex 작업 규칙

이 저장소는 DK-AI-DEV-VM-01의 실제 ASTA 환경이다. 다른 서버의 ASTA/OADT2 작업 트리와 혼동하지 않는다.

## 시작 전

1. 현재 브랜치와 `git status`를 확인한다.
2. 기존 변경을 임의로 되돌리거나 덮어쓰지 않는다.
3. `/opt/select-ai-test/.agent-handoff/CONTEXT.md`가 있으면 먼저 읽고 이전 Hermes/Codex 작업 맥락을 이어간다.
4. 현재 Codex 세션의 기존 대화와 저장소 상태를 함께 근거로 삼는다.

## 작업 중

- 구현 요청은 실제 코드 수정, 관련 테스트 실행, 실행 결과 확인까지 완료한다.
- ASTA 실환경과 로컬/별도 ASTA 환경을 섞지 않는다.
- 인증정보, 토큰, DB 비밀번호, 쿠키를 응답이나 인계 파일에 기록하지 않는다.
- 사용자가 명시하지 않은 배포, 외부 전송, DB 변경은 수행하지 않는다.
- 동시에 다른 Codex 작업이 진행 중인 흔적이 있으면 기존 변경을 보존하고 충돌 가능성을 보고한다.

## 종료 시 인계

작업으로 파일을 변경했거나 중요한 진단을 했다면 `/opt/select-ai-test/.agent-handoff/CONTEXT.md`를 다음 항목으로 갱신한다.

- 요청과 결론
- 변경 파일
- 실행한 테스트와 실제 결과
- 남은 문제/다음 단계
- 관련 커밋(있는 경우)

비밀정보와 긴 원문 로그는 넣지 않는다. 이 파일은 이후 원격 Codex와 Telegram Hermes가 공통 참조한다.
