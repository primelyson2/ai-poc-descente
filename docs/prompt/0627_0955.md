## 튜닝 후 SQL
```sql
WITH filtered_times AS
  (SELECT time_id,
          calendar_year
   FROM DEVDO.TIMES
   WHERE calendar_year IN (1999,
                           2000) ),
     agg_data AS
  (SELECT ft.calendar_year,
          COUNT(*) cnt,
          SUM(s.amount_sold) amt
   FROM DEVDO.SALES s
   JOIN filtered_times ft ON s.time_id = ft.time_id
   GROUP BY ft.calendar_year)
SELECT TO_CHAR(calendar_year) yy,
       cnt,
       amt
FROM agg_data
ORDER BY calendar_year
```

## 상세 분석
튜닝 SQL은 WITH CTE를 활용해 TIMES 테이블의 필터링을 사전 처리하고, SALES와의 JOIN을 한 번으로 통합하여 UNION ALL의 중복 스캔을 제거. XPLAN에서 HASH JOIN(Id 4)의 Starts=1 유지, E-Rows 229K*2 → 459K로 조정되며 A-Rows 480K 처리로 효율화. Buffer Gets 반감은 SALES FULL SCAN(Id 6,8,12 → Id 6 단일)에서 기인, A-Time 0.07s → 0.06s로 CPU_ms 69 → 57ms 단축. 결과 동일성(full_multiset, rows=2 일치) 확인됨.
